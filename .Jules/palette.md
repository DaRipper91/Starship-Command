## 2024-03-06 - Missing ARIA labels pattern in interactive components
**Learning:** Found an accessibility issue pattern where many icon-only buttons across components (like `TerminalPreview`, `FormatEditor`, `FormatSegmentEditor`) were missing `aria-label` and `aria-expanded` attributes. Also learned that when writing tests for components using global contexts (like `ToastContext`), the test must explicitly wrap the component under test in the provider (e.g., `<ToastProvider>`), otherwise hooks like `useToast()` will throw errors and fail the test.
**Action:** Always ensure icon-only buttons have descriptive `aria-label`s and state attributes like `aria-expanded`. When adding or fixing component tests, verify that all context providers required by the component's internal hooks are present in the test render tree.
