# 🤖 Starship Command Enhanced: Master Handbook (v1.0-RC)

> 🎯 **TL;DR:** High-performance, cross-platform Starship theme editor for Web and Android.
> 🎯 _Quick Glance: A visual tool to design Starship prompts with real-time preview, optimized for both desktop browsers and native mobile use._

## 1. User Manual

### Project Overview

Starship Command Enhanced is a professional, visual, no-code editor for creating beautiful [Starship](https://starship.rs) shell prompt themes. It is designed to run as a high-performance Single Page Application (SPA) on the web and as a native Android APK.

### Quick Start

- **Web Editor (Desktop):** Access the browser-based tool (default: `localhost:5173`) to visually configure your prompt.
- **Android APK:** Download `Starship-Command-Enhanced-v1.0-RC-DarkPastel.apk` from the repository root or the GitHub Actions "Artifacts" section.
- **Basic Operation:**
  1.  **Select Modules:** Enable/disable Starship modules (e.g., `directory`, `git_branch`).
  2.  **Configure:** Use the visual UI to set icons, styles, and truncation settings.
  3.  **Preview:** View real-time terminal rendering via the integrated `xterm.js` engine.
  4.  **Export:** Download or copy the generated TOML and place it in `~/.config/starship.toml`.

> 💡 **Expert Opinion:** For the best mobile experience, use the "Auto" layout mode. It intelligently switches to a single-column view with a 5-tab navigation bar optimized for the Pixel 9 and other modern Android devices.

## 2. Technical Manual

### System Architecture

The application employs a **Hybrid Mobile** strategy:

- **Core Frontend:** React 18, TypeScript, Tailwind CSS.
- **State Management:** Zustand (leveraging Zundo for history/undo/redo).
- **Terminal Simulation:** `xterm.js` paired with a custom Starship format parser.
- **Mobile Bridge:** Capacitor wraps the SPA into a native Android container.
- **Optional Backend:** Flask/Python server for advanced color extraction and community sharing.

### Infrastructure & Dependencies

- **Node.js:** v22+ (Required for build and Capacitor synchronization).
- **Java:** OpenJDK 21 (Required for Android APK compilation).
- **Build Tool:** Vite 7.
- **Styling:** Tailwind CSS 3.

> 🧠 **Architectural Note:** The system uses granular Zustand selectors (the "Bolt" pattern) to minimize re-renders. This design is critical for maintaining terminal preview responsiveness on low-power mobile CPUs where full-state subscriptions would cause noticeable input lag.

## 3. Developer Guide

### Code Structure

- `src/`: Main React/TypeScript application logic.
- `android/`: Native Android project files generated by Capacitor.
- `server/`: Python backend logic for color extraction.
- `assets/`: Source SVGs and high-resolution PNGs for branding.

### Development Environment Setup

1.  **Frontend:** `npm install` && `npm run dev`.
2.  **Backend:** `cd server && python3 -m venv venv && pip install -r requirements.txt`.
3.  **Android:** `npm run build && npx cap sync && npx cap open android`.

### Testing & Quality

- **Test Runner:** Vitest with React Testing Library.
- **Execution:** `npm test`.
- **Linting:** `npm run lint` (Enforces strict TypeScript and Prettier rules).

> ⚠️ **Watch Out:** Always run `npx cap sync android` after making changes to the `src/` directory. Failure to sync will result in the native APK running an outdated version of the web bundle.

## 4. Standard Operating Procedures (SOP)

### Workflow: Creating a Theme from a Wallpaper

1.  Navigate to the **Colors** tab (Mobile) or the color sidebar (Desktop).
2.  Select **Upload Image** or provide a direct image URL.
3.  The system uses `node-vibrant` via a Web Worker to extract dominant colors.
4.  Apply the extracted palette to the current theme configuration.

### Workflow: Mobile Reordering

1.  Open the **Modules** tab.
2.  Identify the module to move in the "Active Modules" list.
3.  Tap the **↑ (Up)** or **↓ (Down)** arrow buttons.
    - _Note: Standard Drag-and-Drop is disabled on mobile to prevent accidental touch gestures; use the arrow buttons instead._

### Workflow: Generating a Release APK

1.  Ensure all code is committed and pushed to `main`.
2.  The **GitHub Actions** "Build Android APK" workflow triggers automatically.
3.  Wait for the "success" status (approx. 3-5 minutes).
4.  Download the `starship-command-debug-apk` from the "Artifacts" panel.

## 5. Glossary

- **Starship:** A minimal, blazing-fast, and infinitely customizable prompt for any shell.
- **Capacitor:** A cross-platform app runtime that makes it easy to build web apps that run natively on iOS, Android, and the Web.
- **Zustand:** A small, fast, and scalable bearbones state-management solution using simplified flux principles.
- **xterm.js:** A terminal front-end component written in TypeScript that lets applications provide fully featured terminals in the browser.
- **TOML:** Tom's Obvious, Minimal Language; the configuration file format used by Starship.
